## 发布说明

### JitPack 发布

使用 JitPack 发布新版本：

```bash
# 1. 提交更改
git add .
git commit -m "Release 1.0.7 - Description"

# 2. 推送到远程
git push origin main

# 3. 创建标签
git tag 1.0.7
git push origin 1.0.7

# 4. 访问 JitPack
# https://jitpack.io/#706412584/orangeplayer
```

### 版本命名规则

遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范：

- **主版本号**：不兼容的 API 修改
- **次版本号**：向下兼容的功能新增
- **修订号**：向下兼容的问题修复

示例：`1.2.3`
- `1` = 主版本号
- `2` = 次版本号
- `3` = 修订号

## 贡献指南

欢迎提交 Issue 和 Pull Request！

### 报告 Bug

请在 GitHub Issues 中提供：
- 详细的问题描述
- 复现步骤
- 预期行为 vs 实际行为
- 设备信息和 Android 版本

### 提交功能请求

请在 GitHub Issues 中提供：
- 功能描述
- 使用场景
- 预期的 API 设计

### 提交 Pull Request

1. Fork 项目
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启 Pull Request
